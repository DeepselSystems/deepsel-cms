# Phase 3: Create Theme Skeleton

## Goal
Copy the base theme and create empty shells for all planned files from the design map.

## Process

### Step 1: Copy base theme
```bash
cp -r themes/interlinked themes/<theme-name>
```

### Step 2: Update package.json
Change the `name` field to `@themes/<theme-name>`.

### Step 3: Create Astro entry points
For each page identified in `design-map.md`, create an `.astro` file:

**Required (always create):**
- `Index.astro` — generic page (already exists from copy)
- `Blog.astro` — blog list (already exists)
- `single-blog.astro` — blog post (already exists)
- `404.astro` — not found (already exists)

**Per-design page (create new):**
For each page in the design map with a slug (e.g., slug "about"):
- Create `about.astro` following the same pattern as `Index.astro`
- Import the matching page component (e.g., `PageAbout`)
- These auto-register via `theme_imports.py` and match by slug in `getThemeComponent.ts`

Template for custom page `.astro` file:
```astro
---
import type { PageData } from "@deepsel/cms-utils";
import PageSlug from "./components/PageSlug";
import "./main.css";

interface Props { data: PageData; }
const { data } = Astro.props;
---

<!doctype html>
<html lang={data.lang || "en"}>
  <head>
    <meta charset="utf-8" />
    <title>{data.seo_metadata?.title || ""}</title>
    <meta name="description" content={data.seo_metadata?.description || ""} />
    <meta name="robots" content={data.seo_metadata?.allow_indexing ? "index, follow" : "noindex, nofollow"} />
    <!-- Google Fonts link here -->
  </head>
  <body>
    <PageSlug client:load pageData={data} />
  </body>
</html>
```

### Step 4: Create component shells
For each component in the design map, create an empty TypeScript file in `components/`:

**Required components** (update existing from copy):
- `Page.tsx` — generic fallback
- `BlogList.tsx`
- `BlogPost.tsx`
- `Menu.tsx`
- `Footer.tsx`
- `LangSwitcher.tsx`
- `Sidebar.tsx` (only if sidebar found in design)

**Per-design page components** (create new):
- `Page<Name>.tsx` for each distinct page (e.g., `PageAbout.tsx`, `PageServices.tsx`)

**Section components** (create new if needed):
- `Hero.tsx`, `FeatureGrid.tsx`, etc. — only if the section appears on multiple pages

### Step 5: Copy assets from vibe_figma output

Collect ALL assets generated by vibe_figma and copy them to the theme:

```bash
# Create assets directory
mkdir -p themes/<theme-name>/assets/images/

# Copy all assets from all vibe_figma page outputs
for page_dir in $MIGRATION_DIR/vibe-output/*/assets/; do
  if [ -d "$page_dir" ]; then
    cp -n "$page_dir"* themes/<theme-name>/assets/images/ 2>/dev/null
  fi
done

# List all copied assets
ls -la themes/<theme-name>/assets/images/
```

If multiple pages have assets with the same name but different content, rename with page prefix.

### Step 6: Verify file structure
List all created files and compare against the design map. Every planned component should have a file. Every asset should be copied.

## Rules
- Component names use PascalCase: `PageAbout`, `PageServices`, `FeatureGrid`
- Astro filenames use the exact slug in lowercase: `about.astro`, `services.astro`
- Do NOT create components for pages that don't exist in the Figma design
- Keep all existing config files from the copy (tailwind.config.js, postcss.config.js, tsconfig.json, env.d.ts)
- **Copy ALL assets from vibe_figma output** — these are the actual design images/icons
- Remove default placeholder assets from interlinked (hero.jpg etc.) if they're not in the Figma design

## Output
- Complete file tree in `themes/<theme-name>/` with all shells ready for content
- All assets from vibe_figma copied to `themes/<theme-name>/assets/images/`
